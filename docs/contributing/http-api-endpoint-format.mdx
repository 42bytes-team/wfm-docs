# HTTP API Endpoint Format

Use this page as the rich Markdown/MDX reference for endpoint documentation style. The examples are illustrative format samples, not public API contracts.

## Visual Language

Endpoint pages can stay mostly Markdown, but use MDX components where readability benefits from richer UI.

### Conventions

- Use endpoint headers with method pills, route paths, alternative routes, and metadata chips.
- Keep chips text-first; icons and emoji are visual aids, not the only meaning.
- Use custom callouts or the `Admonition` component for warnings and notes until `:::` parsing is configured.
- Split each endpoint into `Request` and `Response` sections.
- Use `####` headings inside those sections for URL parameters, query parameters, headers, body, fields, constraints, and response fields.
- Use tables for URL parameters, query parameters, headers, request fields, response fields, and model fields.
- Keep JSON examples copyable and syntactically valid.

</Admonition>

## Complex JSON Endpoint

<ApiEndpoint
  method="PATCH"
  path="/v2/example/{id}"
  aliases={["/v2/exampleSlug/{slug}"]}
  summary="Update an example resource and return the updated model."
  badges={[
    {kind: 'auth', icon: '🔒', label: 'Auth required'},
    {kind: 'crossplay', icon: '🌀', label: 'Crossplay aware'},
    {kind: 'i18n', icon: '🇬🇧', label: 'Language aware'},
    {kind: 'accessibility', icon: '♿', label: 'Accessible tables'},
    {kind: 'cache', icon: '⏱', label: '60s cache'},
  ]}
/>

<ApiCallout kind="warning" icon="🔒" title="Requires">

- Authentication
- Scope: `examples.update`

</ApiCallout>

<ApiCallout kind="info" icon="ℹ" title="Response Context">

Response data depends on the `Platform`, `Crossplay`, and `Language` headers. Responses are cached for `60s`.

</ApiCallout>

### Request

#### URL Parameters

| Name | Type | Description |
|---|---|---|
| `id` | <ApiBadge kind="neutral" label="string" /> | `id` field from the linked model. |
| `slug` | <ApiBadge kind="neutral" label="string" /> | `slug` field from the linked model. |

#### Query Parameters

| Name | Type | Description |
|---|---|---|
| `limit` | <ApiBadge kind="neutral" label="int" /> | Maximum number of records. Default: `50`. Min: `1`. Max: `100`. |
| `cursor` | <ApiBadge kind="neutral" label="string" /> | Pagination cursor from a previous response. |

#### Headers

| Name | Type | Description |
|---|---|---|
| `Authorization` | <ApiBadge kind="auth" label="string" /> | JWT access token in the format `Bearer YourJWTAccessToken`. |
| `Language` | <ApiBadge kind="i18n" label="Language" /> | Translation language. Default: `en`. |
| `Platform` | <ApiBadge kind="neutral" label="Platform" /> | Platform context. Default: `pc`. |
| `Crossplay` | <ApiBadge kind="crossplay" label="bool" /> | Crossplay context. Default: `true`. |

#### Body

```json
{
  "name": "Example",
  "visible": true,
  "note": "Optional note"
}
```

#### Body Fields

| Field | Type | Description |
|---|---|---|
| `name` | <ApiBadge kind="neutral" label="string" /> | Display name. Min: `3`. Max: `32`. |
| `visible` | <ApiBadge kind="neutral" label="bool" /> | Whether the resource should be visible. |
| `note` | <ApiBadge kind="neutral" label="string or null" /> | Optional. Note shown only to the owner. |

#### Constraints

- `name` must be unique for the authenticated user.
- `limit` is ignored when `cursor` is present.

### Response

#### Body

```json
{
  "apiVersion": "x.x.x",
  "data": {
    "id": "example-id",
    "name": "Example",
    "visible": true,
    "note": "Optional note"
  },
  "error": null
}
```

#### Data Fields

| Field | Access | Type | Description |
|---|---|---|---|
| `id` |  | <ApiBadge kind="neutral" label="string" /> | Unique example identifier. |
| `name` |  | <ApiBadge kind="neutral" label="string" /> | Display name. |
| `visible` |  | <ApiBadge kind="neutral" label="bool" /> | Whether the resource is visible. |
| `note` | <ApiBadge kind="auth" icon="👤" label="owner" /> | <ApiBadge kind="neutral" label="string or null" /> | Owner-only note. |

## Multipart Endpoint

<ApiEndpoint
  method="POST"
  path="/v2/example/{id}/image"
  summary="Upload an image for an example resource."
  badges={[
    {kind: 'auth', icon: '🔒', label: 'Auth required'},
    {kind: 'firstParty', icon: '◆', label: 'First-party only'},
    {kind: 'danger', icon: '💔', label: 'Restricted surface'},
  ]}
/>

<ApiCallout kind="warning" icon="⚠" title="Restriction">

First-party applications only.

</ApiCallout>

### Request

#### URL Parameters

| Name | Type | Description |
|---|---|---|
| `id` | <ApiBadge kind="neutral" label="string" /> | `id` field from the linked model. |

#### Body

Content-Type: `multipart/form-data`

```http
form-data; name="image"; filename="example.png"
```

#### Body Fields

| Field | Type | Description |
|---|---|---|
| `image` | <ApiBadge kind="neutral" label="file" /> | Image file to upload. |

#### Constraints

- Accepts image formats: `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`, `.bmp`, `.avif`.
- Maximum upload size is `5 MiB`.
- Image is resized to `256x256` pixels, focusing on the center of the image.

### Response

#### Body

```json
{
  "apiVersion": "x.x.x",
  "data": {
    "id": "example-id",
    "imageUrl": "https://example.com/example.png"
  },
  "error": null
}
```

## Orders Top Endpoint Draft

<ApiEndpoint
  method="GET"
  path="/v2/orders/item/{slug}/top"
  aliases={["/v2/orders/itemId/{itemId}/top"]}
  summary="Get the top buy and sell orders for an item from online users."
  badges={[
    {kind: 'crossplay', icon: '🌀', label: 'Crossplay aware'},
    {kind: 'neutral', icon: '⇅', label: 'Custom ranking'},
  ]}
/>

Results depend on the `Platform` and `Crossplay` headers. The response contains up to five `sell` orders and up to five `buy` orders.

Sell orders are ranked from the buyer's point of view: Endo listings first, then better Endo-per-platinum ratio for Endo listings, otherwise lower platinum-per-trade first.

Buy orders are ranked from the seller's point of view: Endo listings first, then lower Endo-per-platinum ratio for Endo listings, otherwise higher platinum-per-trade first.

### Request

#### URL Parameters

| Name | Type | Description |
|---|---|---|
| `slug` | <ApiBadge kind="neutral" label="string" /> | `slug` field from the `Item` or `ItemShort` model. |
| `itemId` | <ApiBadge kind="neutral" label="string" /> | `id` field from the `Item` or `ItemShort` model. |

#### Query Parameters

| Name | Type | Description |
|---|---|---|
| `rank` | <ApiBadge kind="neutral" label="int" /> | Exact item rank. Accepted range: `0` to item max rank. Ignored when the item has no ranks or when `rankLt` is provided. |
| `rankLt` | <ApiBadge kind="neutral" label="int" /> | Match ranks lower than this value. Accepted range: `1` to item max rank. Takes precedence over `rank`. |
| `charges` | <ApiBadge kind="neutral" label="int" /> | Exact remaining charges. Accepted range: `0` to item max charges. Ignored when the item has no charges or when `chargesLt` is provided. |
| `chargesLt` | <ApiBadge kind="neutral" label="int" /> | Match remaining charges lower than this value. Accepted range: `1` to item max charges. Takes precedence over `charges`. |
| `amberStars` | <ApiBadge kind="neutral" label="int" /> | Exact amber star count. Accepted range: `0` to item max amber stars. Ignored when the item has no amber stars or when `amberStarsLt` is provided. |
| `amberStarsLt` | <ApiBadge kind="neutral" label="int" /> | Match amber star counts lower than this value. Accepted range: `1` to item max amber stars. Takes precedence over `amberStars`. |
| `cyanStars` | <ApiBadge kind="neutral" label="int" /> | Exact cyan star count. Accepted range: `0` to item max cyan stars. Ignored when the item has no cyan stars or when `cyanStarsLt` is provided. |
| `cyanStarsLt` | <ApiBadge kind="neutral" label="int" /> | Match cyan star counts lower than this value. Accepted range: `1` to item max cyan stars. Takes precedence over `cyanStars`. |
| `subtype` | <ApiBadge kind="neutral" label="string" /> | Exact item subtype. Must be one of the subtypes supported by the item. Ignored when the item has no subtypes. |

### Response

#### Body

```json
{
  "apiVersion": "x.x.x",
  "data": {
    "sell": [
      {
        "id": "sell-order-id"
      }
    ],
    "buy": [
      {
        "id": "buy-order-id"
      }
    ]
  },
  "error": null
}
```

#### Data Fields

| Field | Type | Description |
|---|---|---|
| `sell` | <ApiBadge kind="neutral" label="[]Order" /> | Up to five sell orders from online users, sorted from the buyer's point of view. |
| `buy` | <ApiBadge kind="neutral" label="[]Order" /> | Up to five buy orders from online users, sorted from the seller's point of view. |

## User Upload Avatar Draft

<ApiEndpoint
  method="POST"
  path="/v2/me/avatar"
  summary="Update your avatar."
  badges={[
    {kind: 'auth', icon: '🔒', label: 'Auth required'},
    {kind: 'neutral', icon: '☁', label: 'Image upload'},
  ]}
/>

The uploaded image is center-cropped to the target aspect ratio, resized to `256x256` pixels, encoded as WebP, and stored as the new avatar.

### Request

#### Body

Content-Type: `multipart/form-data`

```http
form-data; name="avatar"; filename="whatever.png"
```

#### Body Fields

| Field | Type | Description |
|---|---|---|
| `avatar` | <ApiBadge kind="neutral" label="file" /> | Image file to upload as the new avatar. |

#### Constraints

- Accepts image formats: `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`, `.bmp`, `.avif`.
- Maximum upload size is `5 MiB`.

### Response

#### Body

```json
{
  "apiVersion": "x.x.x",
  "data": {
    "id": "user-id",
    "avatarUrl": "https://example.com/avatar.webp"
  },
  "error": null
}
```

### Response

#### Body

```json
{
  "apiVersion": "x.x.x",
```
