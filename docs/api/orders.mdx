# Orders

Endpoints for marketplace orders.

## Get Recent Orders

<ApiEndpoint
  method="GET"
  path="/v2/orders/recent"
  summary="Get the most recent visible orders from non-banned users."
  badges={[
    {kind: 'crossplay', icon: '🌀', label: 'Platform/crossplay aware'},
    {kind: 'cache', icon: '⏱', label: '1m refresh'},
  ]}
/>

Returns up to `500` visible orders created in the last `4h`, sorted by `createdAt` descending. Results include only users currently online.

### Request

#### URL Parameters

None.

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: Order[],
  error: null,
}
```

`data` is an array of [Order](../data-models.mdx#order) models.

## Get Orders By Item

<ApiEndpoint
  method="GET"
  path="/v2/orders/item/{slug}"
  aliases={["/v2/orders/itemId/{itemId}"]}

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: Order[],
  error: null,
}
```

`data` is an array of [Order](../data-models.mdx#order) models.

## Get Top Orders By Item

<ApiEndpoint
  method="GET"
  path="/v2/orders/item/{slug}/top"
  aliases={["/v2/orders/itemId/{itemId}/top"]}
  summary="Get the top buy and sell orders for an item from online users."
  badges={[
    {kind: 'crossplay', icon: '🌀', label: 'Platform/crossplay aware'},
    {kind: 'neutral', icon: '⇅', label: 'Custom ranking'},
  ]}
/>

The response contains up to `5` `sell` orders and up to `5` `buy` orders.

Sell orders are ranked from the buyer's point of view: Endo listings first, then better Endo-per-platinum ratio for Endo listings, otherwise lower platinum-per-trade first.

Buy orders are ranked from the seller's point of view: Endo listings first, then lower Endo-per-platinum ratio for Endo listings, otherwise higher platinum-per-trade first.

### Request

#### URL Parameters

| Name | Type | Description |
|---|---|---|
| `slug` | <ApiBadge kind="neutral" label="string" /> | `slug` field from the [Item](../data-models.mdx#item) model. |
| `itemId` | <ApiBadge kind="neutral" label="string" /> | `id` field from the [Item](../data-models.mdx#item) model. |

#### Query Parameters

| Name | Type | Description |
|---|---|---|
| `rank` | <ApiBadge kind="neutral" label="int" /> | Exact item rank. Accepted range: `0` to item max rank. Ignored when the item has no ranks or when `rankLt` is provided. |
| `rankLt` | <ApiBadge kind="neutral" label="int" /> | Match ranks lower than this value. Accepted range: `1` to item max rank. Takes precedence over `rank`. |
| `charges` | <ApiBadge kind="neutral" label="int" /> | Exact remaining charges. Accepted range: `0` to item max charges. Ignored when the item has no charges or when `chargesLt` is provided. |
| `chargesLt` | <ApiBadge kind="neutral" label="int" /> | Match remaining charges lower than this value. Accepted range: `1` to item max charges. Takes precedence over `charges`. |
| `amberStars` | <ApiBadge kind="neutral" label="int" /> | Exact amber star count. Accepted range: `0` to item max amber stars. Ignored when the item has no amber stars or when `amberStarsLt` is provided. |
| `amberStarsLt` | <ApiBadge kind="neutral" label="int" /> | Match amber star counts lower than this value. Accepted range: `1` to item max amber stars. Takes precedence over `amberStars`. |
| `cyanStars` | <ApiBadge kind="neutral" label="int" /> | Exact cyan star count. Accepted range: `0` to item max cyan stars. Ignored when the item has no cyan stars or when `cyanStarsLt` is provided. |
| `cyanStarsLt` | <ApiBadge kind="neutral" label="int" /> | Match cyan star counts lower than this value. Accepted range: `1` to item max cyan stars. Takes precedence over `cyanStars`. |
| `subtype` | <ApiBadge kind="neutral" label="string" /> | Exact item subtype. Must be one of the subtypes supported by the item. Ignored when the item has no subtypes. |

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: {
    sell: Order[],
    buy: Order[],
  },
  error: null,
}
```

#### Data Fields

| Field | Type | Description |
|---|---|---|
| `sell` | <ApiBadge kind="neutral" label="Order[]" /> | Up to `5` sell orders from online users, sorted from the buyer's point of view. |
| `buy` | <ApiBadge kind="neutral" label="Order[]" /> | Up to `5` buy orders from online users, sorted from the seller's point of view. |

## Create Order

<ApiEndpoint
  method="POST"
  path="/v2/order"
  summary="Create a new order."
  badges={[{kind: 'auth', icon: '🔒', label: 'Auth required'}]}
/>

<ApiCallout kind="warning" icon="🔒" title="Requires">

- Authentication
- Verified account
- User must not be banned

</ApiCallout>

### Request

#### Body

```json
{
  "itemId": "54aae292e7798909064f1575",
  "type": "sell",
  "platinum": 38,
  "quantity": 12,
  "visible": true,
  "perTrade": 6,
  "rank": 5,
  "charges": 3,
  "subtype": "blueprint",
  "amberStars": 3,
  "cyanStars": 3
}
```

#### Body Fields

| Field | Type | Description |
|---|---|---|
| `itemId` | <ApiBadge kind="neutral" label="string" /> | Required. Item ID. Item must exist and be tradable. |
| `type` | <ApiBadge kind="neutral" label="sell or buy" /> | Required. Order type. |
| `platinum` | <ApiBadge kind="neutral" label="int" /> | Required. Price in platinum. Min: `1`. Max: `900000`. |
| `quantity` | <ApiBadge kind="neutral" label="int" /> | Required. Order quantity. Min: `1`. Max: `9999`. |
| `visible` | <ApiBadge kind="neutral" label="bool" /> | Whether the order is visible. Defaults to `false` when omitted. |
| `perTrade` | <ApiBadge kind="neutral" label="int" /> | Required for bulk-tradable items, forbidden otherwise. Min: `1`. Max: `6`. Must divide `quantity`. |
| `rank` | <ApiBadge kind="neutral" label="int" /> | Required when the item supports ranks, forbidden otherwise. Min: `0`. Max: item max rank. |
| `charges` | <ApiBadge kind="neutral" label="int" /> | Required when the item supports charges, forbidden otherwise. Min: `0`. Max: item max charges. |
| `subtype` | <ApiBadge kind="neutral" label="string" /> | Required when the item has subtypes, forbidden otherwise. Must be one of the item's subtypes. |
| `amberStars` | <ApiBadge kind="neutral" label="int" /> | Required when the item supports amber stars, forbidden otherwise. Min: `0`. Max: item max amber stars. |
| `cyanStars` | <ApiBadge kind="neutral" label="int" /> | Required when the item supports cyan stars, forbidden otherwise. Min: `0`. Max: item max cyan stars. |

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: Order,
  error: null,
}
```

`data` is a compact [Order](../data-models.mdx#order) model.

## Close Order

<ApiEndpoint
  method="POST"
  path="/v2/order/{id}/close"
  summary="Close a portion or all of an existing order."
  badges={[{kind: 'auth', icon: '🔒', label: 'Auth required'}]}
/>

If `quantity` is greater than the remaining order quantity, the service closes the remaining quantity. A full close deletes the order; a partial close updates the remaining quantity.

<ApiCallout kind="warning" icon="🔒" title="Requires">

- Authentication
- Verified account
- User must not be banned
- Order owner or role `admin` and higher

</ApiCallout>

### Request

#### URL Parameters

| Name | Type | Description |
|---|---|---|
| `id` | <ApiBadge kind="neutral" label="string" /> | `id` field from the [Order](../data-models.mdx#order) model. |

#### Body

```json
{
  "quantity": 12
}
```

#### Body Fields

| Field | Type | Description |
|---|---|---|
| `quantity` | <ApiBadge kind="neutral" label="int" /> | Units to close. Min: `1`. Max: `9999`. Must be divisible by the order's `perTrade`. |

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: Transaction,
  error: null,
}
```

`data` is a compact [Transaction](../data-models.mdx#transaction) model.
