# Achievements

Endpoints for public achievements and user achievement progress.

## List Achievements

<ApiEndpoint
  method="GET"
  path="/v2/achievements"
  summary="Get all available achievements."
  badges={[{kind: 'i18n', icon: '🇬🇧', label: 'Language aware'}]}
/>

Secret achievements are omitted unless the requester has role `admin`.

### Request

#### URL Parameters

None.

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: Achievement[],
  error: null,
}
```

`data` is an array of [Achievement](../data-models.mdx#achievement) models.

## Get User Achievements

<ApiEndpoint
  method="GET"
  path="/v2/achievements/user/{slug}"
  aliases={["/v2/achievements/userId/{userId}"]}
  summary="Get a user's achievements."
  badges={[{kind: 'i18n', icon: '🇬🇧', label: 'Language aware'}]}
/>

Anonymous users and unrelated users do not receive hidden achievements. The requested user and users with role `moderator` or higher can receive hidden achievements.

### Request

#### URL Parameters

| Name | Type | Description |
|---|---|---|
| `slug` | <ApiBadge kind="neutral" label="string" /> | `slug` field from the [User](../data-models.mdx#user) model. |
| `userId` | <ApiBadge kind="neutral" label="string" /> | `id` field from the [User](../data-models.mdx#user) model. |

#### Query Parameters

| Name | Type | Description |
|---|---|---|
| `featured` | <ApiBadge kind="neutral" label="presence flag" /> | When present, return only achievements where `state.featured` is `true`. The value is ignored, so `?featured=false` still enables the filter. |

### Response

#### Body

```ts
{
  apiVersion: "x.x.x",
  data: Achievement[],
  error: null,
}
```

`data` is an array of [Achievement](../data-models.mdx#achievement) models.
