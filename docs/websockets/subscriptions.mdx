# Subscriptions

Commands and events for realtime marketplace subscriptions.

## Subscribe To New Orders

<WsMessage
  type="Command"
  route="@wfm|cmd/subscribe/newOrders"
  summary="Subscribe to a live feed of newly posted visible orders from online users."
  badges={[{kind: 'neutral', icon: '💬', label: 'Command'}]}
/>

### Client Message

```json
{
  "route": "@wfm|cmd/subscribe/newOrders",
  "id": "message-id",
  "payload": {
    "platform": "pc",
    "crossplay": true
  }
}
```

### Payload Fields

| Field | Type | Description |
|---|---|---|
| `platform` | <ApiBadge kind="neutral" label="Platform" /> | Optional. One of `pc`, `ps4`, `xbox`, `switch`, `mobile`. Omit to subscribe to all platforms. |
| `crossplay` | <ApiBadge kind="neutral" label="bool" /> | Optional. Defaults to `true`. If `platform` is `switch`, the server forces `crossplay` to `false`. |

You can have multiple active subscriptions at the same time, such as one for `pc` and one for `xbox`.

### Success Message

```json
{
  "route": "@wfm|cmd/subscribe/newOrders:ok",
  "id": "message-id"
}
```

### Error Message

```json
{
  "route": "@wfm|cmd/subscribe/newOrders:error",
  "payload": "app.errors.alreadySubscribed",
  "id": "message-id"
}
```

## Unsubscribe From New Orders

<WsMessage
  type="Command"
  route="@wfm|cmd/unsubscribe/newOrders"
  summary="Unsubscribe from a new-order subscription."
  badges={[{kind: 'neutral', icon: '💬', label: 'Command'}]}
/>

You must send the same payload used for subscription. This is required because a client can have multiple subscriptions active simultaneously.

### Client Message

```json
{
  "route": "@wfm|cmd/unsubscribe/newOrders",
  "id": "message-id",
  "payload": {
    "platform": "pc",
    "crossplay": true
  }
}
```

### Payload Fields

| Field | Type | Description |
|---|---|---|
| `platform` | <ApiBadge kind="neutral" label="Platform" /> | Optional. Must match the subscription payload. Omit only to unsubscribe from an all-platform subscription. |
| `crossplay` | <ApiBadge kind="neutral" label="bool" /> | Optional. Defaults to `true`. If `platform` is `switch`, the server forces `crossplay` to `false`. |

### Success Message

```json
{
  "route": "@wfm|cmd/unsubscribe/newOrders:ok",
  "id": "message-id"
}
```

### Error Message

```json
{
  "route": "@wfm|cmd/unsubscribe/newOrders:error",
  "payload": "app.errors.notSubscribed",
  "id": "message-id"
}
```

## New Order Event

<WsMessage
  type="Event"
  route="@wfm|event/subscriptions/newOrder"
  summary="Send a matching newly created visible order to subscribers."
  badges={[{kind: 'neutral', icon: '🔔', label: 'Event'}]}
/>

Sent to matching new-order subscriptions when a visible order from a non-offline user is created.

```ts
{
  route: "@wfm|event/subscriptions/newOrder",
  payload: Order,
}
```

Payload is an [Order](../data-models.mdx#order) shape.

## Not Implemented Yet

The backend currently contains stubs for item and profile order subscriptions, but those commands are not registered as public WebSocket routes. Do not rely on them until they are documented as active.
